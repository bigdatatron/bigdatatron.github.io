DataTron Service API

Introduction

Welcome to DataTron API. The API is the most direct way to access the service.

Background

DataTron API allows to collect, clean, transform and provide data in a queriable format for analysis. The analysis can be used to route business decisions. We also provide basic visualization tools for reporting, business logic deployment platform and data exploration tool.

DataTron is a event-based data platform. Everything in the dataset is a event.

Example

We will consider Wikipedia dataset for all our examples. Everyday people come to Wikipedia and perform the following actions 

a. create new wiki pages [event_name: wikinew]  
b. edit wiki pages [event_name: wikiedit]  
c. comment on wiki pages [event_name: wikicomment]  
d. ....etc

Every action above can be designated as a event

Each event has a set of properties related to itself. The properties can be best presented with a dictionary of key: value pairs

The wikinew event can be described by the following json document

{"user": "test_wiki_user", "create_timestamp": "2003-09-25T10:49:41", "user_hash_id": "11112", ...}

You can send data in this format to DataTron API

In the wikinew event above, the following are the properties, datatypes and values.

Property name Datatype Value
user string "test_wiki_user"
create_timestamp timestamp "2003-09-25T10:49:41"
user_hash_id long "11112"
... ... ...

API

DataTron API is completely REST based framework

The API consists of three categories

Authentication API

Schema API

Data API

For a quick reference of all api calls, click here

Authentication API

Background

DataTron API's provide high security for anyone to interact with the service. For any client to send their data into the service, they need the following

CLIENT_ID
CLIENT_SECRET_KEY

To obtain this information please email harish@datatron.io

To access DatatTron API, you need security token which are obtained by Authentication API. The security tokens expire every few minutes. When expired, you need to obtain a new one by calling Authentication API again.

/api/login

Description
Login using credentials

URL
http://api.datatron.io/login

Method
POST

Parameters
{"client_id": <YOUR_CLIENT_ID>, "client_key": <YOUR_CLIENT_SECRET_KEY>}]}

Returns

{"token": <SECURITY_TOKEN>}

Example
curl -H "Content-Type: application/json" -X POST -d '{"client_id": YOUR_CLIENT_ID, "client_key": YOUR_CLIENT_KEY}' http://api.datatron.io/login

Example Response

{  
  "token": YOUR_SECURITY_TOKEN 
}

In case the security token is expired, you will receive the following response

{
  "message": "Token has expired"
}

When you receive the above message, please call the Authentication API to get a fresh token

Schema API

Background

Schema refers to the metadata for your dataset. In other words, the datatype of each properties in the event

For the wikinew above, we have the following schema

{"properties":[{"name": "user", "dt": "string", "create_timestamp": "timestamp", "user_hash_id": "long"...}]}

/api/createschema

Description
Create schema for an event

URL
http://api.datatron.io/createschema

Method
POST

Parameters
{"properties":[{"name": <property_name_1>, "dt": <datatype>}]}

Returns

{
  "errors": [],   
  "failure": <failure_count>,   
  "success": <success_count>,   
  "warnings": []  
}

Example
curl -H "Content-Type: application/json" -H "Authorization: Bearer <YOUR_SECURITY_TOKEN>" -X POST -d '{"properties":[{"name": "test_property_1", "dt": "long"}]}' http://api.datatron.io/createschema/test_event_name_1

Example Response { "errors": [], "failure": 0, "success": 1, "warnings": [] }

/api/getschema

Description
Get schema for an event

URL
http://api.datatron.io/getschema/<event_name>

Method
GET

Parameters
None

Returns

[{    
  "dt": <datatype>,   
  "name": <failure_count>,  
  ...
}]

Example
curl -H "Content-Type: application/json" -H "Authorization: Bearer <YOUR_SECURITY_TOKEN>" -X GET http://api.datatron.io/getschema/test_event_name_1

Example Response

{"dt": "long", "name": "test_property_1"}

/api/getallproperties

Description
Get all properties for an event

URL
http://api.datatron.io/getallproperties/<event_name>

Method
GET

Parameters
None

Returns

[{    
  <property_1>,   
  <property_2>,
  ...
}]

Example
curl -H "Content-Type: application/json" -H "Authorization: Bearer <YOUR_SECURITY_TOKEN>" -X GET http://api.datatron.io/getallproperties/test_event_name_1

Example Response

["test_property_1"]

/api/getdatatype

Description
Get datatype for a property in an event

URL
http://api.datatron.io/getdatatype/<event_name>/<property_name>

Method
GET

Parameters
None

Returns

[<datatype>]

Example
curl -H "Content-Type: application/json" -H "Authorization: Bearer <YOUR_SECURITY_TOKEN>" -X GET http://api.datatron.io/getdatatype/test_event_name_1/test_property_1

Example Response

["long"]

GET

Parameters

Returns

Example
curl -H "Content-Type: application/json" -H "Authorization: Bearer <YOUR_SECURITY_TOKEN>" -X GET http://api.datatron.io/getallevents

Data API

Background

Send datasets for the event

/api/events

Description
Send data for event

URL
http://api.datatron.io/events/<event_name>

Method
POST

Parameters 

[
{
  <property_name_1>: <property_value_a>, 
  <property_name_2>: <property_value_b>,
  <property_name_3>: <property_value_c>...
},
{
  <property_name_1>: <property_value_d>, 
  <property_name_2>: <property_value_e>,
  <property_name_3>: <property_value_f>...
},...
]

Returns

{
  "errors": [],   
  "failure": <failure_count>,   
  "success": <success_count>,   
  "warnings": []  
}

Example
curl -H "Content-Type: application/json" -H "Authorization: Bearer <YOUR_SECURITY_TOKEN>" -X POST -d '{"data":[{"test_property_1": "1111"}]}' http://api.datatron.io/events/test_event_name_1

Example Response { "errors": [], "failure": 0, "success": 1, "warnings": [] }
